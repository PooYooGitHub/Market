---

### 测试 2: 用户注册
1. 点击导航栏的"注册"按钮，或访问 http://localhost:5173/register
2. 填写注册信息：
   ```
   用户名: testuser001
   密码: 123456
   确认密码: 123456
   昵称: 测试用户1（可选）
   手机号: 13800138000（可选）
   邮箱: test@example.com（可选）
   ```
3. 点击"注册"按钮

**预期结果**: 
- ✅ 如果后端服务运行中：注册成功，弹出提示，跳转到登录页
- ❌ 如果后端未启动：显示网络错误提示

**常见错误**:
- "用户名必须是4-20位字母、数字或下划线" - 用户名格式错误
- "两次密码输入不一致" - 密码不匹配
- "手机号格式不正确" - 手机号不符合规范
- "用户名已存在" - 该用户名已被注册（换一个用户名）

---

### 测试 3: 用户登录
1. 在登录页输入刚才注册的账号：
   ```
   用户名: testuser001
   密码: 123456
   ```
2. 点击"登录"按钮

**预期结果**: 
- ✅ 登录成功，弹出"登录成功"提示
- ✅ 跳转到首页

**常见错误**:
- "用户名或密码错误" - 检查输入是否正确
- "登录已过期" - Token失效，重新登录

## 🐛 调试工具

### 查看请求
1. 打开浏览器开发者工具（F12）
2. 切换到 Network（网络）标签
3. 执行操作（如登录）
4. 查看请求和响应：
   - Request Headers: 是否包含 `satoken`
   - Response: 查看返回的数据和状态码

### 查看本地存储
1. 开发者工具 → Application（应用）
2. 左侧 Local Storage → http://localhost:5173
3. 查看：
   - `token`: 登录令牌
   - `userInfo`: 用户信息（JSON字符串）

### 查看 Console
1. 开发者工具 → Console（控制台）
2. 查看错误信息和日志输出

## 🔥 常见问题

### Q1: 页面无法访问
**检查**:
- 开发服务器是否启动？
- 端口 5173 是否被占用？
- 浏览器地址是否正确？

### Q2: 请求失败
**检查**:
- 后端服务是否启动？
- API 网关地址是否正确？（http://localhost:9000）
- Network 面板查看具体错误

### Q3: 登录后刷新页面又变成未登录
**检查**:
- localStorage 中是否有 token？
- token 是否正确保存？
- 路由守卫是否正常工作？

### Q4: 样式显示异常
**解决**:
- 清除浏览器缓存（Ctrl+Shift+Delete）
- 硬刷新页面（Ctrl+F5）
