## 功能说明

### 1. 商品图片上传（ProductForm.vue）

**位置**：发布商品页面 `/product/publish` 或编辑商品页面 `/product/edit/:id`

**功能特性**：
- ✅ 支持批量上传（最多9张）
- ✅ 文件格式：JPG、PNG、GIF、WEBP
- ✅ 单张大小限制：5MB
- ✅ 实时预览
- ✅ 上传进度显示
- ✅ 可删除已上传图片
- ✅ 自动验证文件类型和大小

**测试步骤**：

#### 测试1：单张图片上传
1. 登录账号
2. 点击导航栏"发布商品"
3. 点击"添加图片"按钮（+号）
4. 选择1张图片文件
5. 观察上传进度和结果

**预期结果**：
- 选择图片后立即显示本地预览
- 显示"上传中..."状态
- 上传成功后显示完整图片
- 图片显示正常，可以删除

#### 测试2：批量上传（多张）
1. 在发布商品页面
2. 点击"添加图片"
3. 按住 Ctrl（Windows）或 Cmd（Mac）选择多张图片（如5张）
4. 观察批量上传过程

**预期结果**：
- 所有选择的图片同时显示上传状态
- 逐个上传完成
- 显示计数：已上传 X/9 张图片

#### 测试3：超出数量限制
1. 已上传5张图片
2. 尝试再次选择6张图片

**预期结果**：
- 弹出提示："最多上传9张图片，当前已有5张，只能再上传4张"
- 上传操作被阻止

#### 测试4：文件大小限制
1. 准备一张超过5MB的图片
2. 选择该图片上传

**预期结果**：
- 弹出提示："图片大小不能超过 5MB，当前文件大小为 X.XX MB"
- 上传被阻止

#### 测试5：文件格式验证
1. 选择非图片文件（如 .txt, .pdf）
2. 尝试上传

**预期结果**：
- 弹出提示："不支持的文件格式，请上传 JPG、PNG、GIF、WEBP 格式的图片"
- 上传被阻止

#### 测试6：删除图片
1. 上传3张图片
2. 点击第2张图片右上角的"×"按钮

**预期结果**：
- 第2张图片被删除
- 剩余2张图片
- 计数更新为：已上传 2/9 张图片

#### 测试7：完整发布流程
1. 填写商品信息：
   - 标题：测试商品123
   - 分类：选择任意分类
   - 价格：99.90
   - 描述：这是测试商品
2. 上传3张图片
3. 点击"立即发布"

**预期结果**：
- 提交成功
- 跳转到商品详情页
- 详情页正确显示3张图片
- 图片可以轮播查看

## 常见问题排查

### 问题1：上传失败 - 401 Unauthorized

**原因**：Token 未携带或已过期

**解决方法**：
1. 确认已登录
2. 检查 localStorage 中是否有 `satoken`
3. 尝试重新登录

```javascript
// 浏览器控制台检查
localStorage.getItem('satoken')
```

### 问题2：上传失败 - 500 Internal Server Error

**原因**：File 服务或 MinIO 未启动

**解决方法**：
1. 检查 Docker 容器状态
```bash
docker ps
```
2. 重启 MinIO 容器
```bash
docker restart <minio-container-id>
```
3. 检查 File 服务日志

### 问题3：图片上传成功但显示不出来

**原因1**：MinIO 桶权限未设置为公开读

**解决方法**：
1. 访问 MinIO 控制台：`http://localhost:9001`
2. 登录（admin / 3083446265@qq.com）
3. 选择桶（如 market-product）
4. 设置 Access Policy 为 `Public`

**原因2**：图片URL格式错误

**解决方法**：
检查返回的URL格式是否正确：
```
正确: http://localhost:9000/market-product/abc123.jpg
错误: /market-product/abc123.jpg
错误: localhost:9000/market-product/abc123.jpg
```

### 问题4：图片加载慢或超时

**原因**：MinIO 资源不足或网络问题

** 解决方法**：
1. 检查 MinIO 容器资源使用情况
2. 检查图片文件大小
3. 调整 MinIO 配置

### 问题5：选择文件后无反应

**原因**：JavaScript 错误或网络问题

**解决方法**：
1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签页的错误信息
3. 查看 Network 标签页的请求状态
