## 1. 概述

### 1.1 模块说明

用户模块提供用户注册、登录、信息管理等核心功能，是整个系统的基础模块。

### 1.2 服务地址

- **网关地址**: `http://localhost:9000`
- **用户服务**: `http://localhost:8101` (内部服务，前端不直接访问)
- **所有前端请求都通过网关访问**

### 1.3 认证机制

- **Token 类型**: Bearer Token (Sa-Token)
- **Token 传递方式**: 
  - HTTP Header: `satoken: <token值>`
  - 或: `Authorization: Bearer <token值>`
- **Token 有效期**: 30天
- **Token 存储建议**: localStorage 或 sessionStorage

## 2. 通用说明

### 2.1 请求头 (Request Headers)

#### 公共接口（无需登录）
```http
Content-Type: application/json
```

#### 需要登录的接口
```http
Content-Type: application/json
satoken: <登录后获取的token>
```

### 2.2 响应格式 (Response Format)

所有接口统一返回以下格式：

```json
{
  "code": 200,          // 状态码：200成功，其他失败
  "message": "操作成功",  // 提示信息
  "data": {}            // 业务数据，可能是对象、数组或null
}
```

### 2.3 HTTP 状态码

| 状态码 | 说明 | 处理建议 |
|--------|------|---------|
| 200 | 成功 | 正常处理 data 数据 |
| 401 | 未登录或 Token 失效 | 跳转到登录页 |
| 403 | 无权限 | 提示用户权限不足 |
| 500 | 服务器错误 | 提示"服务异常，请稍后重试" |

### 2.4 业务状态码 (Response.code)

| code | 说明 |
|------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或登录失效 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 3. 接口列表

### 3.1 用户注册

#### 基本信息

- **接口地址**: `/api/user/register`
- **请求方式**: `POST`
- **是否需要登录**: ❌ 否（白名单接口）
- **接口说明**: 用户注册新账号

#### 请求参数 (Request Body)

```json
{
  "username": "testuser",           // 必填，4-20位字母、数字或下划线
  "password": "123456",             // 必填，6-20位
  "confirmPassword": "123456",      // 必填，必须与password一致
  "nickname": "测试用户",            // 可选，昵称
  "phone": "13800138000",           // 可选，手机号，需符合格式
  "email": "test@example.com"       // 可选，邮箱，需符合格式
}
```

#### 请求参数说明

| 参数名 | 类型 | 必填 | 说明 | 校验规则 |
|--------|------|------|------|---------|
| username | String | 是 | 用户名 | 4-20位字母、数字或下划线 |
| password | String | 是 | 密码 | 6-20位 |
| confirmPassword | String | 是 | 确认密码 | 必须与password一致 |
| nickname | String | 否 | 昵称 | 如不填，默认为用户名 |
| phone | String | 否 | 手机号 | 11位手机号格式 |
| email | String | 否 | 邮箱 | 标准邮箱格式 |

#### 响应示例

**成功响应**:
```json
{
  "code": 200,
  "message": "注册成功",
  "data": 1001           // 返回用户ID
}
```

**失败响应**:
```json
{
  "code": 400,
  "message": "用户名已存在",
  "data": null
}
```

#### 错误场景

| 错误信息 | 原因 |
|---------|------|
| 用户名不能为空 | username 未传或为空 |
| 用户名必须是4-20位字母、数字或下划线 | username 格式错误 |
| 密码长度必须在6-20位之间 | password 长度不符 |
| 两次密码输入不一致 | password ≠ confirmPassword |
| 用户名已存在 | 该用户名已被注册 |
| 手机号已被注册 | 该手机号已被使用 |
| 手机号格式不正确 | phone 格式错误 |
| 邮箱格式不正确 | email 格式错误 |

#### 前端示例代码

```javascript
// 使用 axios
async function register(formData) {
  try {
    const response = await axios.post('http://localhost:9000/api/user/register', {
      username: formData.username,
      password: formData.password,
      confirmPassword: formData.confirmPassword,
      nickname: formData.nickname,
      phone: formData.phone,
      email: formData.email
    });
    
    if (response.data.code === 200) {
      console.log('注册成功，用户ID:', response.data.data);
      // 跳转到登录页
      router.push('/login');
    } else {
      alert(response.data.message);
    }
  } catch (error) {
    console.error('注册失败:', error);
    alert('注册失败，请稍后重试');
  }
}
```

### 3.2 用户登录

#### 基本信息

- **接口地址**: `/api/user/login`
- **请求方式**: `POST`
- **是否需要登录**: ❌ 否（白名单接口）
- **接口说明**: 用户登录获取 Token

#### 请求参数 (Request Body)

```json
{
  "username": "testuser",     // 必填，用户名
  "password": "123456"        // 必填，密码
}
```

#### 请求参数说明

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | String | 是 | 用户名 |
| password | String | 是 | 密码（明文） |

#### 响应示例

**成功响应**:
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "tokenType": "Bearer",
    "expiresIn": 2592000,      // Token有效期（秒），30天
    "userInfo": {
      "id": 1001,
      "username": "testuser",
      "nickname": "测试用户",
      "avatar": "https://example.com/avatar.jpg",
      "phone": "13800138000",
      "email": "test@example.com",
      "status": 1,
      "createTime": "2026-02-14T10:00:00"
    }
  }
}
```

**失败响应**:
```json
{
  "code": 400,
  "message": "用户名或密码错误",
  "data": null
}
```

#### 响应数据说明

**LoginVO 对象**:

| 字段 | 类型 | 说明 |
|------|------|------|
| token | String | 访问令牌，需保存到本地 |
| tokenType | String | Token类型，固定为 "Bearer" |
| expiresIn | Long | Token有效期（秒），30天=2592000秒 |
| userInfo | UserVO | 用户信息对象 |

**UserVO 对象**:

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 用户ID |
| username | String | 用户名 |
| nickname | String | 昵称 |
| avatar | String | 头像URL |
| phone | String | 手机号 |
| email | String | 邮箱 |
| status | Integer | 状态：1-正常，0-禁用 |
| createTime | String | 注册时间 |

#### 前端示例代码

```javascript
// 登录并保存 Token
async function login(username, password) {
  try {
    const response = await axios.post('http://localhost:9000/api/user/login', {
      username,
      password
    });
    
    if (response.data.code === 200) {
      const { token, userInfo } = response.data.data;
      
      // 保存 Token 到 localStorage
      localStorage.setItem('token', token);
      
      // 保存用户信息
      localStorage.setItem('userInfo', JSON.stringify(userInfo));
      
      console.log('登录成功');
      // 跳转到首页
      router.push('/home');
    } else {
      alert(response.data.message);
    }
  } catch (error) {
    console.error('登录失败:', error);
    alert('登录失败，请稍后重试');
  }
}
```

### 3.3 获取当前用户信息

#### 基本信息

- **接口地址**: `/api/user/info`
- **请求方式**: `GET`
- **是否需要登录**: ✅ 是
- **接口说明**: 获取当前登录用户的详细信息

#### 请求头 (Request Headers)

```http
Content-Type: application/json
satoken: <登录后的token>
```

#### 请求参数

无需传递参数，Gateway 会自动从 Token 中解析用户ID

#### 响应示例

**成功响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1001,
    "username": "testuser",
    "nickname": "测试用户",
    "avatar": "https://example.com/avatar.jpg",
    "phone": "13800138000",
    "email": "test@example.com",
    "status": 1,
    "createTime": "2026-02-14T10:00:00"
  }
}
```

**失败响应**:
```json
{
  "code": 401,
  "message": "未登录或登录已过期",
  "data": null
}
```

```json
{
  "code": 400,
  "message": "用户不存在",
  "data": null
}
```

#### 前端示例代码

```javascript
// 获取当前用户信息
async function getCurrentUserInfo() {
  try {
    const token = localStorage.getItem('token');
    
    const response = await axios.get('http://localhost:9000/api/user/info', {
      headers: {
        'satoken': token
      }
    });
    
    if (response.data.code === 200) {
      return response.data.data;
    } else if (response.data.code === 401) {
      // Token 失效，跳转登录页
      localStorage.removeItem('token');
      router.push('/login');
    } else {
      alert(response.data.message);
    }
  } catch (error) {
    console.error('获取用户信息失败:', error);
  }
}
```

  try {
    const token = localStorage.getItem('token');

}

### 3.5 修改密码

#### 基本信息

- **接口地址**: `/api/user/change-password`
- **请求方式**: `PUT`
- **是否需要登录**: ✅ 是
- **接口说明**: 修改当前用户的密码

#### 请求头 (Request Headers)

```http
Content-Type: application/json
satoken: <登录后的token>
```

#### 请求参数 (Request Body)

```json
{
  "oldPassword": "123456",        // 必填，旧密码
  "newPassword": "newpass123",    // 必填，新密码
  "confirmPassword": "newpass123" // 必填，确认新密码
}
```

#### 请求参数说明

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| oldPassword | String | 是 | 旧密码 |
| newPassword | String | 是 | 新密码，6-20位 |
| confirmPassword | String | 是 | 确认新密码，必须与newPassword一致 |

#### 响应示例

**成功响应**:
```json
{
  "code": 200,
  "message": "密码修改成功，请重新登录",
  "data": "密码修改成功，请重新登录"
}
```

**失败响应**:
```json
{
  "code": 400,
  "message": "两次密码输入不一致",
  "data": null
}
```

```json
{
  "code": 400,
  "message": "密码修改失败",
  "data": null
}
```

#### 错误场景

| 错误信息 | 原因 |
|---------|------|
| 旧密码不能为空 | oldPassword 未传或为空 |
| 新密码不能为空 | newPassword 未传或为空 |
| 确认密码不能为空 | confirmPassword 未传或为空 |
| 两次密码输入不一致 | newPassword ≠ confirmPassword |
| 密码修改失败 | 旧密码错误 |

#### 前端示例代码

```javascript
// 修改密码
async function changePassword(oldPassword, newPassword, confirmPassword) {
  try {
    const token = localStorage.getItem('token');
    
    const response = await axios.put('http://localhost:9000/api/user/change-password', 
      {
        oldPassword,
        newPassword,
        confirmPassword
      },
      {
        headers: {
          'satoken': token
        }
      }
    );
    
    if (response.data.code === 200) {
      alert('密码修改成功，请重新登录');
      // 清除本地 Token
      localStorage.removeItem('token');
      localStorage.removeItem('userInfo');
      // 跳转到登录页
      router.push('/login');
    } else {
      alert(response.data.message);
    }
  } catch (error) {
    console.error('修改密码失败:', error);
    alert('修改密码失败，请稍后重试');
  }
}
```

## 4. 数据模型

interface Result<T> {

## 6. 前端集成示例

### 6.1 Axios 拦截器配置

```javascript
// axios-config.js
import axios from 'axios';
import router from './router';

// 创建 axios 实例
const service = axios.create({
  baseURL: 'http://localhost:9000',
  timeout: 10000
});

// 请求拦截器：自动添加 Token
service.interceptors.request.use(
  config => {
    const token = localStorage.getItem('token');
    if (token) {
      config.headers['satoken'] = token;
    }
    return config;
  },
  error => {
    console.error('请求错误:', error);
    return Promise.reject(error);
  }
);

// 响应拦截器：统一处理错误
service.interceptors.response.use(
  response => {
    const res = response.data;
    
    // code 不是 200 则认为失败
    if (res.code !== 200) {
      // 401 未登录
      if (res.code === 401) {
        alert('登录已过期，请重新登录');
        localStorage.removeItem('token');
        localStorage.removeItem('userInfo');
        router.push('/login');
        return Promise.reject(new Error(res.message || '未登录'));
      }
      
      // 其他错误
      alert(res.message || '操作失败');
      return Promise.reject(new Error(res.message || 'Error'));
    } else {
      return res;
    }
  },
  error => {
    console.error('响应错误:', error);
    
    if (error.response) {
      switch (error.response.status) {
        case 401:
          alert('登录已过期，请重新登录');
          localStorage.removeItem('token');
          router.push('/login');
          break;
        case 403:
          alert('无权限访问');
          break;
        case 404:
          alert('请求的资源不存在');
          break;
        case 500:
          alert('服务器错误，请稍后重试');
          break;
        default:
          alert('网络错误，请检查网络连接');
      }
    } else {
      alert('网络错误，请检查网络连接');
    }
    
    return Promise.reject(error);
  }
);

export default service;
```

## 7. 注意事项

### 7.1 Token 管理

1. **Token 存储位置**
   - 推荐：`localStorage` （持久化，刷新页面不丢失）
   - 可选：`sessionStorage` （会话期，关闭标签页即失效）

2. **Token 自动携带**
   - 使用 Axios 拦截器自动添加到请求头
   - Header 名称：`satoken`

3. **Token 失效处理**
   - 监听 401 响应
   - 清除本地 Token
   - 跳转到登录页

### 7.2 安全建议

1. **密码处理**
   - 前端传输明文密码
   - 后端自动加密存储
   - 不要在前端加密（无意义）

2. **敏感信息**
   - 不要在 localStorage 存储敏感信息
   - Token 已足够，不需要存密码

3. **HTTPS**
   - 生产环境必须使用 HTTPS
   - 防止 Token 被中间人截获

### 7.3 开发调试

1. **本地开发环境**
   - Gateway: `http://localhost:9000`
   - 前端: `http://localhost:5173` (Vite默认)

2. **跨域问题**
   - Gateway 已配置 CORS
   - 本地开发无需配置代理

3. **调试工具**
   - Chrome DevTools → Network
   - 查看请求/响应详情
   - 检查 Header 中的 Token

## 8. 常见问题 FAQ

### Q1: 登录后为什么访问其他接口还是 401？

**A**: 检查以下几点：
1. Token 是否正确保存到 localStorage
2. 请求头是否携带 Token（Header: `satoken`）
3. Token 是否已过期（30天有效期）

### Q2: 如何判断用户是否已登录？

**A**: 
```javascript
function isLoggedIn() {
  const token = localStorage.getItem('token');
  return token !== null && token !== '';
}
```

### Q3: 注册时提示"用户名已存在"怎么办？

**A**: 换一个用户名，该用户名已被其他用户注册。

### Q4: 修改密码后需要重新登录吗？

**A**: 是的，密码修改成功后，原 Token 仍然有效，但建议重新登录以确保安全。

### Q5: 可以同时在多个设备登录吗？

**A**: 可以，每次登录生成独立的 Token，互不影响。

### Q6: Token 有效期是多久？

**A**: 30天（2592000秒），过期后需要重新登录。
