## 📋 常见问题

### Q1: 启动时提示 RocketMQ 连接失败？
**A**: 这是正常的警告，不影响功能。系统会自动降级。

### Q2: 消息发送后没有推送？
**A**: 检查：
1. 接收者是否在线？（WebSocket 连接）
2. 浏览器控制台是否有错误？
3. 后端日志是否显示发送成功？

### Q3: 需要启动 RocketMQ 吗？
**A**: 
- **开发测试**: 不需要 ✅
- **生产环境**: 建议启动 ✅
- **毕业设计**: 不需要 ✅

## 💡 修复说明

### 修改的文件：
1. ✅ `MessageServiceImpl.java` - 添加 `@Autowired(required = false)`
2. ✅ `application.yml` - 延长超时时间
3. ✅ `RocketMQConfig.java` - 添加配置类

### 修复原理：
```
发送消息
    ├─ 保存到数据库 ✅
    ├─ WebSocket 推送 ✅
    └─ RocketMQ 推送 (可选)
         ├─ 成功 → 记录日志
         └─ 失败 → 降级（不影响前面步骤）✅
```
