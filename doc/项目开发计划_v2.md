## 一、项目模块结构

### 1.1 最终模块结构

```
Market (父工程)
├── market-common                 # 公共模块
│   ├── constant/                 # 常量
│   ├── enums/                    # 枚举
│   ├── exception/                # 异常
│   ├── result/                   # 统一响应
│   └── util/                     # 工具类
│
├── market-api                    # Feign 接口模块
│   ├── market-api-user           # 用户服务接口
│   ├── market-api-product        # 商品服务接口
│   └── market-api-trade          # 交易服务接口
│
├── market-gateway                # 网关服务 [端口: 9000]
│   ├── filter/                   # 认证过滤器 ⭐
│   └── config/                   # 配置
│
└── market-service                # 业务服务
    ├── market-service-user       # 用户服务 [端口: 8101]
    ├── market-service-product    # 商品服务 [端口: 8102]
    └── market-service-trade      # 交易服务 [端口: 8103]
```

### 1.2 关于 Auth 服务

**❌ 不再单独部署 Auth 服务**

| 原方案 | 新方案 |
|--------|--------|
| Auth 服务负责登录 | User 服务负责登录 |
| Auth 服务生成 Token | User 服务生成 Token |
| 每次请求调用 Auth 验证 | Gateway 本地验证（读 Redis） |
| 服务间依赖复杂 | 依赖清晰简单 |

## 二、服务职责划分

### 2.1 Gateway（网关）

| 职责 | 说明 |
|------|------|
| 路由转发 | 根据路径转发到对应服务 |
| 统一认证 | 验证 Token，提取 userId |
| 跨域处理 | 统一 CORS 配置 |
| 限流熔断 | Sentinel（可选） |

**不负责**：
- ❌ 业务逻辑
- ❌ 数据库操作

### 2.2 User 服务

| 职责 | 说明 |
|------|------|
| 用户注册 | 密码加密、重复检查 |
| 用户登录 | 验证密码、生成 Token |
| 用户信息 | 查询、修改用户信息 |
| Feign 接口 | 供其他服务调用 |

**数据库**：`market_user`

### 2.3 Product 服务

| 职责 | 说明 |
|------|------|
| 商品发布 | 创建、编辑商品 |
| 商品查询 | 列表、搜索、详情 |
| 分类管理 | 商品分类 CRUD |
| 收藏管理 | 用户收藏商品 |

**数据库**：`market_product`

### 2.4 Trade 服务

| 职责 | 说明 |
|------|------|
| 购物车 | 添加、删除、查询 |
| 订单管理 | 创建、查询、取消 |
| 订单状态 | 状态流转 |

**数据库**：`market_trade`

## 七、启动顺序

```
1. 基础设施
   ├── MySQL (3306)
   ├── Redis (6379)
   └── Nacos (8849)

2. 业务服务（顺序无关）
   ├── market-service-user (8101)
   ├── market-service-product (8102)
   └── market-service-trade (8103)

3. 网关
   └── market-gateway (9000)
```

### 快速启动脚本

```powershell
# start-all.ps1
Write-Host "Starting Market Platform..."

# 1. 检查基础设施
docker-compose up -d mysql redis nacos

# 2. 等待服务就绪
Start-Sleep -Seconds 10

# 3. 启动业务服务
Start-Process powershell -ArgumentList "-Command cd market-service\market-service-user; mvn spring-boot:run"
Start-Process powershell -ArgumentList "-Command cd market-service\market-service-product; mvn spring-boot:run"

# 4. 启动网关
Start-Sleep -Seconds 30
Start-Process powershell -ArgumentList "-Command cd market-gateway; mvn spring-boot:run"

Write-Host "All services started!"
```

## 九、注意事项

### 9.1 开发规范

1. **Controller 返回值**：统一使用 `Result<T>`
2. **异常处理**：抛出 `BusinessException`
3. **日志规范**：使用 `@Slf4j`
4. **代码风格**：遵循阿里巴巴 Java 开发手册

### 9.2 Feign 调用规范

1. Feign 接口返回 `Result<T>`
2. Feign Controller 路径以 `/feign/` 开头
3. 调用时检查 `result.getCode() == 200`

### 9.3 数据库规范

1. 每个服务只访问自己的数据库
2. 禁止跨库 JOIN
3. 关联数据通过 Feign 获取
