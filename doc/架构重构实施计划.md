## 一、当前问题分析

### 1.1 现有架构的问题

| 问题 | 描述 | 影响 |
|------|------|------|
| Auth 服务职责不清 | Auth 通过 Feign 调用 User，每次请求都要调用 | 性能差，延迟高 |
| 认证流程复杂 | 前端→Gateway→Auth→User | 链路长，容易出错 |
| 职责重叠 | Auth 和 User 都处理登录相关逻辑 | 代码冗余，维护困难 |
| Gateway 缺少认证 | Gateway 没有认证过滤器 | 所有服务都要自己验证 |

### 1.2 目标架构

```
前端 → Gateway（认证） → 业务服务
           ↓
        Redis（验证Token）
```

```java
// UserServiceImpl.java 新增方法

@Override
public LoginVO login(LoginRequest request) {
    // 1. 查询用户
    User user = getByUsername(request.getUsername());
    if (user == null) {
        throw new BusinessException("用户名或密码错误");
    }
    
    // 2. 验证密码
    if (!BCrypt.checkpw(request.getPassword(), user.getPassword())) {
        throw new BusinessException("用户名或密码错误");
    }
    
    // 3. 检查状态
    if (user.getStatus() != 1) {
        throw new BusinessException("账号已被禁用");
    }
    
    // 4. 生成 Token
    StpUtil.login(user.getId());
    String token = StpUtil.getTokenValue();
    
    // 5. 返回结果
    return LoginVO.builder()
        .token(token)
        .userId(user.getId())
        .username(user.getUsername())
        .nickname(user.getNickname())
        .avatar(user.getAvatar())
        .build();
}
```

### 3.2 Gateway - 认证过滤器

```java
// AuthGlobalFilter.java

@Component
@Slf4j
public class AuthGlobalFilter implements GlobalFilter, Ordered {

    @Autowired
    private StringRedisTemplate redisTemplate;
    
    // 白名单
    private static final List<String> WHITE_LIST = Arrays.asList(
        "/api/user/login",
        "/api/user/register",
        "/api/product/list",
        "/api/product/detail",
        "/api/product/category"
    );

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        String path = exchange.getRequest().getURI().getPath();
        
        // 1. 白名单放行
        if (isWhiteList(path)) {
            return chain.filter(exchange);
        }
        
        // 2. 获取 Token
        String token = exchange.getRequest().getHeaders().getFirst("satoken");
        if (StringUtils.isEmpty(token)) {
            return unauthorized(exchange, "请先登录");
        }
        
        // 3. 从 Redis 验证 Token
        String userId = redisTemplate.opsForValue().get("satoken:login:token:" + token);
        if (StringUtils.isEmpty(userId)) {
            return unauthorized(exchange, "登录已过期，请重新登录");
        }
        
        // 4. 将 userId 放入 Header
        ServerHttpRequest request = exchange.getRequest().mutate()
            .header("X-User-Id", userId)
            .build();
        
        return chain.filter(exchange.mutate().request(request).build());
    }
    
    private boolean isWhiteList(String path) {
        return WHITE_LIST.stream().anyMatch(path::startsWith);
    }
    
    private Mono<Void> unauthorized(ServerWebExchange exchange, String message) {
        ServerHttpResponse response = exchange.getResponse();
        response.setStatusCode(HttpStatus.UNAUTHORIZED);
        response.getHeaders().setContentType(MediaType.APPLICATION_JSON);
        
        String body = "{\"code\":401,\"message\":\"" + message + "\"}";
        DataBuffer buffer = response.bufferFactory().wrap(body.getBytes(StandardCharsets.UTF_8));
        return response.writeWith(Mono.just(buffer));
    }
    
    @Override
    public int getOrder() {
        return -100;
    }
}
```

### 4.2 接口测试

```bash
# 1. 注册
curl -X POST http://localhost:9000/api/user/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"123456","confirmPassword":"123456"}'

# 2. 登录
curl -X POST http://localhost:9000/api/user/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"123456"}'

# 3. 访问受保护接口（带 Token）
curl -X GET http://localhost:9000/api/user/info \
  -H "satoken: {从登录获取的token}"

# 4. 访问白名单接口（无需 Token）
curl -X GET http://localhost:9000/api/product/list
```

## 五、风险评估

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Redis 不可用 | 无法验证 Token | 确保 Redis 高可用 |
| Token 被盗用 | 账号被劫持 | 添加 IP 绑定（可选） |
| 白名单配置错误 | 安全漏洞 | 仔细检查白名单 |
