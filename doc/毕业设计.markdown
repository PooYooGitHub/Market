| 1．  本课题的研究目的及意义  随着高校在校生规模的不断扩大和消费观念的转变，二手物品交易需求日益增长。传统的校园跳蚤市场多依赖线下或社交平台进行，存在信息不透明、交易效率低、信任机制缺失、纠纷处理困难等问题。当前校园二手交易平台多基于单体架构开发，存在扩展性差、维护困难、高并发处理能力不足等局限性。  本课题旨在设计并实现一套基于 SpringCloud 微服务架构的校园跳蚤市场平台，覆盖“商品发布→智能搜索→即时聊天→信用评价→交易仲裁→数据运营”全链路流程。系统采用前后端分离架构，前端基于 Vue3+Pinia+PWA 实现响应式与离线体验，后端采用  SpringCloud Alibaba 2022 套件（Nacos、Gateway、Sentinel、Seata、RocketMQ）构建高可用、可扩展的微服务体系，业务服务划分为用户、商品、交易、消息、信用、文件等多个微服务。  **研究目标包括：**  （1）构建高可用、易扩展的微服务架构，提升系统的并发处理能力和服务治理水平；   （2）实现全链路交易闭环，提升用户体验与交易安全性；   （3）引入信用评价与仲裁机制，建立可信交易环境；   （4）通过数据运营模块，为平台运营提供决策支持。  **研究意义：**  本课题不仅在技术上探索微服务架构在校园电商场景中的应用，还具有重要的实践意义，可推动校园资源循环利用、促进绿色消费，提升校园二手交易的信息化与智能化水平，为构建可信、高效、便捷的校园交易生态提供解决方案。 |

| 2．  已了解的本课题国内外研究现状  **国内现状：**  国内校园二手交易平台多以微信公众号、小程序或独立APP形式存在，如“闲鱼校园版”、“转转校园”等，功能主要集中在商品发布与浏览，但在即时通讯、信用体系、纠纷处理等方面较为薄弱。多数平台仍采用单体或简单分布式架构，面临性能瓶颈和扩展难题。近年来，部分高校尝试自建二手平台，但普遍存在用户活跃度低、功能单一、技术支持不足等问题。  **国外现状：**  国外校园二手平台如“Facebook Marketplace”、“UniMarket”等，在用户体验、社交整合、信用机制方面较为成熟。部分平台引入区块链技术进行信用存证，或集成第三方支付与物流服务，构建更完善的交易生态。在架构上，越来越多平台采用微服务与容器化部署，提升系统的弹性和可维护性。  **技术趋势：**  微服务架构已成为中大型电商系统的首选，SpringCloud 生态在服务注册、配置管理、流量控制、分布式事务等方面提供成熟解决方案。前后端分离、PWA、即时通讯、信用风控等技术也逐渐成为校园交易平台的标配。 |
| 3．  本课题的研究内容  本课题将设计并实现一个基于 SpringCloud 微服务架构的校园跳蚤市场平台，系统主要包括以下模块：  **系统角色：**  · **普通用户：** 注册登录、发布商品、搜索商品、即时聊天、下单购买、评价、申请仲裁等。  · **管理员：** 用户管理、商品审核、纠纷仲裁、数据统计、系统配置等。  **核心功能模块：**  （1）**用户服务：** 负责用户注册、登录、个人信息管理、信用分管理等。   （2）**商品服务：** 支持商品发布、分类展示、智能搜索。   （3）**交易服务：** 实现购物车、订单管理   （4）**消息服务：** 基于 WebSocket 实现用户间即时通讯，支持文字、图片消息。   （5）**信用服务：** 构建用户信用评价体系，支持信用分计算、评价展示、  （6）**仲裁服务：** 提供交易纠纷提交、处理、裁决流程，支持管理员介入。   （7）**文件服务：** 统一管理图片、文档等资源，支持上传、下载、预览。   （8）**数据看板：** 为管理员提供交易统计、用户行为分析、平台运营数据可视化。 |
