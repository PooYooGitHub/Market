# 校园跳蚤市场平台 - 项目开发计划

> 基于 SpringCloud Alibaba 微服务架构的校园二手交易平台

---

## 一、项目模块结构

### 1. 模块总览

```
Market (父工程)
├── market-common                 # 公共模块（工具类、常量、异常、通用实体）
├── market-api                    # 服务接口模块（Feign接口、DTO）
│   ├── market-api-user          # 用户服务接口
│   ├── market-api-product       # 商品服务接口
│   ├── market-api-trade         # 交易服务接口
│   ├── market-api-message       # 消息服务接口
│   ├── market-api-credit        # 信用服务接口
│   ├── market-api-arbitration   # 仲裁服务接口
│   └── market-api-file          # 文件服务接口
├── market-gateway                # API网关服务
├── market-auth                   # 认证授权服务
├── market-service                # 业务服务模块
│   ├── market-service-user      # 用户服务
│   ├── market-service-product   # 商品服务
│   ├── market-service-trade     # 交易服务
│   ├── market-service-message   # 消息服务
│   ├── market-service-credit    # 信用服务
│   ├── market-service-arbitration # 仲裁服务
│   ├── market-service-file      # 文件服务
│   └── market-service-admin     # 管理后台服务（数据看板）
└── market-frontend               # 前端工程（Vue3）
```

### 2. 各模块职责说明

| 模块名称 | 类型 | 职责描述 |
|---------|------|---------|
| **market-common** | 公共模块 | 通用工具类、常量定义、异常处理、通用响应封装、分页工具等 |
| **market-api** | 接口模块 | 定义各服务的Feign远程调用接口和DTO对象，供服务间调用 |
| **market-gateway** | 网关服务 | 统一入口、路由转发、鉴权过滤、限流熔断、跨域处理 |
| **market-auth** | 认证服务 | JWT令牌生成、用户登录认证、权限校验、OAuth2集成 |
| **market-service-user** | 业务服务 | 用户注册、登录、个人信息管理、角色权限管理 |
| **market-service-product** | 业务服务 | 商品CRUD、分类管理、商品审核、智能搜索(ES) |
| **market-service-trade** | 业务服务 | 购物车、订单管理、支付流程、交易状态管理 |
| **market-service-message** | 业务服务 | WebSocket即时通讯、系统通知、消息记录存储 |
| **market-service-credit** | 业务服务 | 信用分计算、用户评价、信用等级管理 |
| **market-service-arbitration** | 业务服务 | 纠纷提交、仲裁流程、裁决记录管理 |
| **market-service-file** | 业务服务 | 文件上传下载、图片处理、OSS/MinIO对接 |
| **market-service-admin** | 业务服务 | 数据统计、运营看板、系统配置、日志管理 |

## 四、数据库设计概览

### 用户服务数据库 (market_user)

```sql
-- 用户表
t_user (id, username, password, nickname, avatar, phone, email, status, create_time, update_time)

-- 用户角色表
t_user_role (id, user_id, role_id)

-- 角色表
t_role (id, role_name, role_code, description)
```

### 商品服务数据库 (market_product)

```sql
-- 商品表
t_product (id, user_id, title, description, price, original_price, category_id, status, view_count, create_time)

-- 商品图片表
t_product_image (id, product_id, image_url, sort)

-- 分类表
t_category (id, parent_id, name, sort, icon)
```

### 交易服务数据库 (market_trade)

```sql
-- 购物车表
t_cart (id, user_id, product_id, quantity, selected)

-- 订单表
t_order (id, order_no, buyer_id, seller_id, product_id, amount, status, create_time)

-- 订单状态记录表
t_order_log (id, order_id, status, operator, remark, create_time)
```

### 消息服务数据库 (market_message)

```sql
-- 聊天消息表
t_chat_message (id, sender_id, receiver_id, content, type, read_status, create_time)

-- 系统通知表
t_notification (id, user_id, title, content, type, read_status, create_time)
```

### 信用服务数据库 (market_credit)

```sql
-- 信用分表
t_credit_score (id, user_id, score, level, update_time)

-- 评价表
t_evaluation (id, order_id, evaluator_id, target_id, score, content, create_time)
```

### 仲裁服务数据库 (market_arbitration)

```sql
-- 仲裁申请表
t_arbitration (id, order_id, applicant_id, respondent_id, reason, evidence, status, result, handler_id, create_time)

-- 仲裁记录表
t_arbitration_log (id, arbitration_id, operator_id, action, remark, create_time)
```

### 文件服务数据库 (market_file)

```sql
-- 文件信息表
t_file_info (id, file_name, file_path, file_size, file_type, uploader_id, create_time)
```

## 七、开发规范

### 代码规范
- 遵循阿里巴巴Java开发手册
- 统一使用 Lombok 减少样板代码
- RESTful API 设计规范
- 统一异常处理和响应格式

### Git规范
- 分支策略：main(生产) / develop(开发) / feature-xxx(功能)
- 提交信息格式：`[类型] 简短描述`
  - 类型：feat(功能) / fix(修复) / docs(文档) / refactor(重构)

### 接口规范
- 统一响应格式：`{ code, message, data, timestamp }`
- 使用 Swagger/Knife4j 生成API文档
- 接口版本控制：`/api/v1/xxx`
