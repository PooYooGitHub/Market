# 校园跳蚤市场平台 - 项目开发计划

> 基于 SpringCloud Alibaba 微服务架构的校园二手交易平台  
> ⚡ **当前架构**: 轻量级微服务（3核心服务）- 适合开发环境和内存受限场景

---

## 一、项目模块结构

### 1. 当前实际模块结构（轻量级架构）

```
Market (父工程)
├── market-common                 # 公共模块（工具类、常量、异常、通用实体）
├── market-api                    # 服务接口模块（Feign接口、DTO）
│   ├── market-api-user          # 用户服务接口
│   ├── market-api-product       # 商品服务接口
│   ├── market-api-trade         # 交易服务接口
│   ├── market-api-message       # 消息服务接口（预留）
│   ├── market-api-credit        # 信用服务接口（预留）
│   ├── market-api-arbitration   # 仲裁服务接口（预留）
│   └── market-api-file          # 文件服务接口（预留）
├── market-gateway                # API网关服务 ✅ [端口: 8080]
├── market-auth                   # 认证授权服务 ✅ [端口: 8081]
└── market-service                # 业务服务模块
    ├── market-service-core      # 核心业务服务 ✅ [端口: 9001] (整合user+product+trade)
    ├── market-service-user      # 用户服务（代码被core整合）
    ├── market-service-product   # 商品服务（代码被core整合）
    └── market-service-trade     # 交易服务（代码被core整合）
```

### 2. 轻量级架构 vs 完整架构对比

| 架构类型 | 需要启动的服务数 | 内存占用 | 适用场景 |
|---------|----------------|---------|---------|
| **轻量级架构（当前）** | 3个 | ~1.5GB | 开发环境、内存受限、演示 |
| 完整微服务架构 | 10+个 | ~4GB+ | 生产环境、高并发、分布式部署 |

### 3. 核心服务说明

| 服务 | 端口 | 职责 | 启动优先级 |
|-----|------|-----|-----------|
| **market-gateway** | 8080 | 统一入口、路由转发、鉴权过滤、跨域处理 | 1 |
| **market-auth** | 8081 | JWT令牌生成、用户登录认证、权限校验 | 2 |
| **market-service-core** | 9001 | 整合用户/商品/交易核心业务 | 3 |

### 4. 核心服务整合原理

`market-service-core` 通过组件扫描整合了三个业务模块：

```java
@SpringBootApplication(scanBasePackages = {
    "org.shyu.marketservicecore",
    "org.shyu.marketserviceuser",
    "org.shyu.marketserviceproduct",
    "org.shyu.marketservicetrade"
})
@MapperScan({
    "org.shyu.marketserviceuser.mapper",
    "org.shyu.marketserviceproduct.mapper",
    "org.shyu.marketservicetrade.mapper"
})
```

## 三、快速启动指南

### 1. 启动基础设施

```powershell
# 启动 MySQL、Redis、Nacos (Docker Compose)
docker-compose up -d
```

### 2. 初始化数据库

```sql
-- 依次执行以下SQL脚本
doc/SQL/init_schema.sql
doc/SQL/mysql-schema.sql
doc/SQL/sample_data.sql
```

### 3. 编译项目

```powershell
mvn clean package -DskipTests
```

### 4. 启动服务（按顺序）

```powershell
# 终端1 - 启动网关
cd market-gateway\target
java -jar market-gateway-1.0-SNAPSHOT.jar

# 终端2 - 启动认证服务
cd market-auth\target
java -jar market-auth-1.0-SNAPSHOT.jar

# 终端3 - 启动核心业务服务
cd market-service\market-service-core\target
java -jar market-service-core-1.0-SNAPSHOT.jar
```

### 5. 验证服务

| 验证项 | 地址 | 预期结果 |
|-------|------|---------|
| Nacos控制台 | http://localhost:8849/nacos | 可登录 (nacos/nacos) |
| API网关 | http://localhost:8080 | 服务运行中 |
| 服务注册 | Nacos服务列表 | 3个服务已注册 |

```sql

t_user (id, username, password, nickname, avatar, phone, email, status, create_time, update_time)

```

## 八、开发规范

### 代码规范
- 遵循阿里巴巴Java开发手册
- 统一使用 Lombok 减少样板代码
- RESTful API 设计规范
- 统一异常处理和响应格式

### Git规范
- 分支策略：main(生产) / develop(开发) / feature-xxx(功能)
- 提交信息格式：`[类型] 简短描述`
  - 类型：feat(功能) / fix(修复) / docs(文档) / refactor(重构)

### 接口规范
- 统一响应格式：`{ code, message, data, timestamp }`
- 使用 Knife4j 生成API文档
- 接口路径：`/api/user/**`, `/api/product/**`, `/api/trade/**`
